# 環境構築手順（Docker版）

## 前提条件
- Docker Desktop がインストールされていること
- Git がインストールされていること

## セットアップ手順

### 1. リポジトリのクローン
```bash
git clone [リポジトリURL]
cd [プロジェクトディレクトリ]
```

### 2. 環境変数の設定
```bash
# 環境変数ファイルをコピー
cp .env.example .env
```

### 3. .envファイルの編集
テキストエディタで`.env`ファイルを開き、以下の値を設定：
.envの内容はDiscordで共有しときます
```env
# Supabase設定
NEXT_PUBLIC_SUPABASE_URL=[共有されたSupabase URL]
NEXT_PUBLIC_SUPABASE_ANON_KEY=[共有されたANONキー]

# Backend用
SUPABASE_URL=[共有されたSupabase URL]
SUPABASE_KEY=[共有されたSERVICE_ROLEキー]
DATABASE_URL=[共有されたデータベースURL]
SECRET_KEY=[任意の文字列]
```

### 4. Dockerコンテナの起動
```bash
# バックグラウンドで起動
docker-compose up -d

# ログを見ながら起動（デバッグ時）
docker-compose up
```

### 5. アプリケーションへのアクセス
起動後、以下のURLでアクセスできます：

- **フロントエンド**: http://localhost:3000
- **バックエンドAPI**: http://localhost:8000
- **APIドキュメント**: http://localhost:8000/docs
